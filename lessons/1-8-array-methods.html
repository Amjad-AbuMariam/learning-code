1-8 — Array Methods

~60 minutes · Week 1, Day 3 · Source: curriculum array-methods.md

Why this matters: map, filter, and reduce are the vocabulary of every job interview and half of every React component you'll write in this course.

Idea 1 — Three families

The curriculum's method list is long; it collapses into three families. Learn the middle one deeply, recognize the others:

FamilyMethodsWhat they do
Mutatorspush pop shift unshiftChange the array in place (end / start)
Transformersmap filter reduce (+ forEach)Run your function per element; return a new array/value, original untouched
Askersincludes some every indexOf findIndex Array.isArrayAnswer a question — boolean or index. Recognize; look up on the cheat sheet

Idea 2 — The transformers, one sentence each

const nums = [3, 5, 4];

nums.map(n => n * 2)              // [6, 10, 8]   same length, each element transformed
nums.filter(n => n % 2 === 0)     // [4]          keeps elements passing your test
nums.reduce((acc, n) => acc + n, 0) // 12         folds everything into one value
nums.forEach(n => console.log(n)) // prints each; returns nothing — side effects only
The coursebook writes these with function (num) { return … }; today we write arrow functions — same behavior, less noise. Both forms will appear in code you read.

Do it for real (~35 min)

In the browser console. Type each, predicting the output first.

  1. Mutators — run one line at a time, and type arr after each to watch it change: 
    const arr = [1, 2, 3]
arr.pop()        // removes the LAST element
arr.push(3, 4)   // adds to the end
arr.shift()      // removes the FIRST element
arr.unshift(1)   // adds to the start
    Success check: arr ends as [1, 2, 3, 4] and you can say which end each method touched.
  2. Prove transformers don't mutate: 
    const nums = [3, 5, 4]
nums.map(n => n * 2)   // → [6, 10, 8]
nums                   // → [3, 5, 4]
    You should see nums unchanged — the doubled array was returned, not stored.
  3. Chain two transformers — fill in both blanks to get the doubled evens: 
    const six = [1, 2, 3, 4, 5, 6]
six.filter(/* keep the evens */).map(/* double them */)
    Check: exactly [4, 8, 12].
  4. The curriculum's reduce challenge — complete the reducer so it finds the maximum: 
    function maxOf(nums) {
  return nums.reduce(/* your reducer here */);
}
maxOf([3, 41, 7])   // must return 41
maxOf([-5, -2])     // must return -2
    (Hint: the accumulator carries the biggest-so-far.)
  5. One asker — predict before you run it: 
    [1, 2, 4].some(n => n % 2 === 1)
    Check: true — the 1 passes the is-odd test.

Check yourself — predict the output

const words = ["git", "css", "flexbox", "html"];
const result = words.filter(w => w.length === 3).map(w => w.toUpperCase());
Reveal

["GIT", "CSS"] — filter keeps the three-letter words (order preserved), map uppercases the survivors. words itself is unchanged.
