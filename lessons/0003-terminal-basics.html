Lesson 3 — Terminal Basics

~45 minutes · Curriculum: Week 1 (terminal workshop) · No prior Linux assumed

Why this matters for the mission: from Week 4 onward (Node, Express, PostgreSQL) the terminal is your workbench — servers, databases, and git all live there. Developers who navigate it fluently are visibly faster in pair programming and interviews. You've already used it for git without realizing how much you know.

Idea 1 — Terminal vs shell

The terminal is just the window. The shell (yours is bash) is the program inside it that reads your command, runs it, and prints the result. Every command has the same shape:

ls        -l        ~/learning-code
│         │         │
command   flag      argument
(what)    (how)     (on what)

Idea 2 — You are always somewhere

The filesystem is one big tree starting at the root /. Your shell always sits at one spot in that tree (the working directory), and every relative path is read from there. Three symbols do most of the work: ~ is your home (/home/amjad), . is here, .. is one level up. A path starting with / is absolute (works from anywhere); anything else is relative (depends on where you are). Full command table: terminal cheat sheet.

Press Tab after every few characters of a path. It autocompletes, and — more importantly — if nothing completes, your path is wrong before you run the command. This one habit makes beginners look experienced.

Do it for real (~25 min)

Type each of these — no copy-paste, typing builds the muscle memory. Predict the output before pressing Enter.

  1. Orient: open a terminal. Run pwd, then ls, then ls -la. What extra rows did -a reveal? (Files starting with . are hidden — you'll meet .git this way.)
  2. Navigate: cd / then ls — you're at the root of the whole system. Now get home three ways: cd /home/amjad (absolute), then cd / and cd home/amjad (relative), then cd / and just cd.
  3. Build a tree: from home: mkdir -p terminal-practice/notes, cd terminal-practice, touch notes/day1.md, ls notes.
  4. Write & read without an editor: echo "terminal day one" > notes/day1.md, then cat notes/day1.md. Append with echo "second line" >> notes/day1.md and cat again — note > replaces, >> appends.
  5. Copy, rename, move: cp notes/day1.md notes/backup.md, mv notes/backup.md notes/day1-backup.md (mv = rename), mv notes/day1-backup.md . (move it here).
  6. Search: cd ~/learning-code then grep -r "flexbox" --include="*.html" . -l — which lesson files mention flexbox? Then find . -name "*.md" | head.
  7. History: press ↑ a few times. Then Ctrl-R and type grep — it finds your earlier command. Run it again with Enter.
  8. Clean up (carefully): cd ~, rm -r terminal-practice. Read twice before Enter: rm has no trash can and no undo. Never run rm -r on a path you haven't just ls'd.
