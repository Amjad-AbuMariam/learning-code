Week 1 — Fundamentals & Toolkit

Curriculum source: coursebook/week-1 · ~20 hours including the project

What this week leaves you with

From the curriculum's learning outcomes: semantic HTML that conveys meaning, flexbox and responsive/mobile-first layout, accessibility and the tools that check it, the terminal, the git/GitHub flow, JS fundamentals (types, variables, operators, arrays, objects), and a first designed-then-built project — including the UX outcomes (user stories, Figma first contact) folded into the project's design step. Week 2 (testing with npm/Jest) assumes the terminal and git are muscle memory — that's why they sit mid-week here, before the project.

Prerequisites

  • freeCodeCamp basics done: HTML, CSS, JS fundamentals, DOM manipulation (lessons 1-7 to 1-9 refresh rather than teach; 1-7 has a skip checkpoint).
  • A Linux machine with a browser, a text editor (VS Code recommended), git installed, and a GitHub account. Nothing else — lesson 1-2 needs no terminal, and lesson 1-4 assumes zero terminal experience.

Sequencing notes

  • Kept from the curriculum, flagged for awareness: HTML/CSS (Day 1) come before the terminal and git (Day 2). Lessons 1-2 and 1-3 are therefore deliberately editor-and-browser only; version control is applied to that work retroactively from 1-6 onward. If you prefer tools-first, doing 1-4 → 1-5 before 1-2 → 1-3 also works — nothing in Day 1 depends on Day 2.
  • Compressed: the curriculum's Day 3 "Intro to JS" assumes zero JavaScript; this track's prerequisites don't. 1-7 carries a skip checkpoint instead of three sessions.
  • Reordered inside Day 4: the curriculum runs the DOM morning challenge first, then objects and operators; 1-9 flips that so the DOM task can use the just-refreshed objects and operators on your own page (morning challenges assume a cohort warm-up format).
  • Adapted for solo: pair workshops (git-workflow-for-two, pair programming methods, peer code review) run with the agent as partner — instructions live in HTML comments inside 1-5 and 1-10. The Day 3 UX/Figma session is folded into the project's design-first step.
  • Dropped as cohort-only: welcome talks, ice-breakers, safeguarding, morning-inspiration slots.

How feedback works this week

Every lesson footer asks two questions (followability 1–5, stuck-points); tell your answers to your teacher (the agent), which records them in feedback/week-1.md. Low scores rewrite the lesson before the next student sees it.
