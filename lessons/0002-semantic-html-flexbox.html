Lesson 2 — Semantic HTML & Flexbox

~45 minutes · Curriculum: Week 1 (Fundamentals & Toolkit)

Why this matters for the mission: every project week from here on starts with a page skeleton and a layout. Semantic HTML is also an interview topic (accessibility questions are common), and Flexbox is the workhorse layout tool — the curriculum's Week 1 project is graded partly on both.

Idea 1 — HTML that says what it means

freeCodeCamp taught you the tags; the professional habit is choosing them for meaning. A <div class="header"> and a <header> render identically — but only one is announced to screen-reader users, skimmed by search engines, and instantly legible to the next developer. The skeleton to internalize is in the cheat sheet: header / nav / main / section / article / aside / footer, with exactly one <main> per page.

Idea 2 — Flexbox is two axes, four knobs

Everything in flexbox follows from one fact: the container has a main axis (set by flex-direction) and a cross axis perpendicular to it. justify-content distributes along main; align-items aligns along cross. Flip the direction and their visual roles swap.

Build it (~25 min)

In your week-0/ folder, create a personal profile page — this becomes the seed of your portfolio:

  1. Branch first (Lesson 1 habit): git switch -c feat/profile-page.
  2. Create week-0/index.html with the full semantic skeleton: header with your name and a nav (3 links), main with one section ("About") and two article cards ("Projects"), and a footer.
  3. Style with flexbox only, in a <style> block or linked css file:
    • nav: links in a row, spaced with gap, pushed to one side with justify-content
    • the two project cards: side by side, equal width (flex: 1), wrapping on narrow screens (flex-wrap)
    • body: make the footer stick to the bottom on short pages (hint: flex-direction: column; min-height: 100vh and let main grow)
  4. Commit (imperative mood, say why), push, open a PR, merge it, and delete the remote branch this time.
