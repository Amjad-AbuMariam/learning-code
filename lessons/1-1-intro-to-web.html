1-1 — How the Web Works

~30 minutes · Week 1, Day 1 · Source: curriculum intro-to-web.md

Why this matters: every project in this course is a website. Thirty minutes of shared vocabulary now — front end, back end, static, dynamic — makes every later lesson read faster.

Idea 1 — The web is documents linked by URLs

The internet is the network; the web is what runs on it: billions of pages written in HTML, delivered to your browser over HTTP, and connected to each other by links. A website is a collection of such pages under one domain name, served by a web server.

Two kinds of website:

StaticPages stored ready-made on the server, sent as-is. Pure HTML + CSS. Your week-1 pages are static.
DynamicPages generated at request time by code (JavaScript, Python, PHP…), usually from a database. Amazon's product pages are dynamic.

Idea 2 — Front end, back end, full stack

The front end is everything the browser shows: HTML structure, CSS appearance, JavaScript behavior. The back end is the code and data on the server that feeds it. A full-stack developer works on both — that is where this course is headed (HTML/CSS → JS → Node/Express → PostgreSQL → React).

Do it for real (~10 min)

  1. Open any website you like in your browser. Press F12 (or right-click → Inspect). You should now see a panel with an Elements tab showing the page's HTML.
  2. In the Elements tab, click a few arrows to unfold the tree. You should be able to find the text of the page inside HTML tags — this is exactly what the server sent (plus what JavaScript changed).
  3. Open the Console tab in the same panel and type 1 + 1, then Enter. You should see 2. Remember this console — lesson 1-8 uses it as a JavaScript playground.
