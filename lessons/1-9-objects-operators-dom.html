1-9 — Objects, Operators & the DOM

~60 minutes · Week 1, Day 4 · Source: curriculum Day 4 (JavaScript objects, comparison & logical operators, DOM morning challenge)

Why this matters: objects are how every API this course touches represents data; the operators decide every if; and the DOM exercise reconnects your freeCodeCamp knowledge to the profile page you actually own.

Idea 1 — Objects: labeled data

const project = {
  name: "Profile page",
  tech: ["html", "css"],
  finished: false,
  describe() { return `${this.name} (${this.tech.join(", ")})`; },
};

project.name          // "Profile page"   — dot access
project["name"]       // same, bracket access — needed when the key is in a variable
project.describe()    // "Profile page (html, css)"

Idea 2 — Comparisons that don't lie

Always compare with === / !== (no type conversion). The loose forms == / != convert types before comparing ("5" == 5 is true) — a classic bug source you'll see in older code but shouldn't write. Logical operators: && (both), || (either), ! (not) — and they return operands, not booleans, which is why name || "anonymous" works as a fallback.

Part A — console (~15 min).

  1. Build a me object — same shape as the project example in Idea 1 — with your name, a skills array, and a yearsOfCS number. Check: me.skills.length returns the count you expect.

Part B — DOM challenge on your page (~20 min). The curriculum's Day 4 sets a DOM manipulation challenge; here it targets your own page. The four DOM calls below are freeCodeCamp material — they're also in the JavaScript cheat sheet's DOM section; skim it now if they feel far away. Open ~/code/week-1/index.html in the browser, work in the console first, then make it permanent.

  1. Grab your heading: const h1 = document.querySelector("h1"), then h1.textContent. You should see your name.
  2. In the console, add a live skill counter: 
    const skills = ["html", "css", "flexbox", "git"];
const p = document.createElement("p");
p.textContent = `Currently practicing ${skills.length} skills: ${skills.join(", ")}`;
document.querySelector("#about").append(p);
    You should see the paragraph appear in your About section. Refresh — it's gone: console changes aren't saved.
  3. Make it permanent: in a terminal, cd ~/code/week-1 (the repo you published in 1-6), start a branch with git switch -c add-skill-counter, then in your editor put the same code in a <script> tag just before </body>. Refresh. You should see it survive reloads now.
  4. Commit, push, open a PR, merge, sync — the full 1-5 loop, from memory, in the same terminal (still in ~/code/week-1). Success check: the PR shows exactly one changed file, and after syncing, git branch lists only main.

Check yourself — predict the output

const user = { name: "", city: "Gaza" };
console.log(user.name || "anonymous");
console.log(user.name === undefined);
Reveal

"anonymous" — empty string is falsy, so || falls through. Then false — name exists and is "", which is not undefined. (A key that's truly absent, like user.email, would give undefined.)
