Idea 1 — Values have types; variables are labels

JavaScript's primitive types you'll actually meet daily: string, number, boolean, undefined (nothing assigned yet), null (deliberately nothing). Everything else — objects and arrays — is compared by reference: two arrays with the same content are still two different arrays. typeof value asks a value its type.

Idea 2 — const by default

Modern JS declares with const unless reassignment is genuinely needed, then let. You will still see var in older curriculum snippets — it's the legacy form with looser scoping; don't write new code with it. Note the nuance the console showed you: const stops reassignment of the label, not mutation of the object it points to — const arr = []; arr.push(1) is legal.

Do it for real (~15 min)

In the console:

  1. Declare const profile = "your name" then try profile = "other". You should see a TypeError naming the constant.
  2. Run const skills = ["html", "css"]; skills.push("git"); skills. You should see all three items — mutation through a const label works.
  3. Run typeof null. You should see "object" — a famous historical bug kept for compatibility; recognize it so it never confuses you.
