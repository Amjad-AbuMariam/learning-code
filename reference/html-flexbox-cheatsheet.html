Semantic HTML & Flexbox

Page skeleton and layout, at a glance.

Semantic page skeleton

<body>
  <header>        site banner: logo, title
    <nav>         primary navigation links
  </header>
  <main>          THE content — exactly one per page
    <section>     thematic grouping, deserves a heading
    <article>     self-contained: post, card, comment
    <aside>       tangential: sidebar, related links
  </main>
  <footer>        copyright, contact, small print
</body>

Rule of thumb: reach for <div> only when no semantic element fits — it means nothing to screen readers or search engines.

Flexbox — container properties

PropertyValuesControls
display: flex—turns element into a flex container
flex-directionrow · row-reverse · column · column-reversewhich way the main axis runs
justify-contentflex-start · center · flex-end · space-between · space-around · space-evenlydistribution along the main axis
align-itemsstretch · flex-start · center · flex-end · baselinealignment along the cross axis
flex-wrapnowrap · wrap · wrap-reverseoverflow onto new lines
gaplengthspace between items (use instead of margins)

Flexbox — item properties

PropertyMeaning
flex-growshare of leftover space this item absorbs (0 = don't grow)
flex-shrinkhow much it gives up when space is tight (default 1)
flex-basisstarting size before grow/shrink
flex: 1shorthand: grow 1, shrink 1, basis 0 — equal-width columns
align-selfoverride align-items for one item

Mental model

Main axis = the direction items flow (set by flex-direction). Cross axis = perpendicular. justify-content works on main, align-items on cross — when direction flips to column, their visual effects swap too.
